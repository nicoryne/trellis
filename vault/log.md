## [2026-05-15] ingest | Classifier-gated chat: knowledge vs. conversational paths + universal Gemini retry

16 commits ahead of remote. Major chat architecture revision: every message now routes through a **Gemini Flash classifier** ([[chat-query-classifier]]) that returns `knowledge` (existing RAG) or `conversational` (no retrieval). The SSE protocol now leads with a `kind` event so the frontend can gate the [[query-overlay-animation]] correctly before any cited-nodes payload arrives. Conversational replies stream from `gemini-2.5-flash` under a tone-matched prompt with a **hard rule against substantive legal claims from training data** (rule 6 in `prompts/conversational.md`) ‚Äî the privilege boundary survives the new path. Created three concept pages: [[chat-query-classifier]], [[conversational-chat-path]], [[gemini-retry-backoff]]. Multi-turn conversation memory (last 4 turns, server cap 8) shipped via `buildHistoryPayload` in `apps/web/src/api/chat.ts` ‚Äî this was explicitly listed as **out of MVP** in PRD ¬ß2.7 but landed for demo polish; conversational follow-ups can re-quote `[node_id]` citations from the prior assistant turn and they resolve through the new `allCitedNodeIds` union memo in [ChatView](apps/web/src/views/chat/ChatView.tsx) as clickable chips.

**Universal Gemini retry/backoff** ([[gemini-retry-backoff|withGeminiRetry]] in `services/gemini-retry.ts`): every Gemini SDK call now goes through this wrapper ‚Äî capture (transcribe + vision), organize, redaction (Pass 2 + preserve), embedding, classifier, RAG stream, conversational stream. Exponential backoff with full jitter, per-attempt SDK `timeout`, retryable-error classification (408/425/429/5xx + abort + network-layer + status-in-message-string). Streaming connect retries only ‚Äî mid-stream chunks aren't safely resumable. Classifier uses tight `maxAttempts: 2, timeoutMs: 8_000` because chat latency is a hero-moment risk and the fallback (return `knowledge`) is safe. **Lifts the prior fragility note** in [[gemini]] about "no retry, no timeout, transient 5xx surfaces as 500."

**RAG confidence thresholds re-calibrated** (`services/rag.ts:64‚Äì81` comment): empirical measurement on `gemini-embedding-001` distributions showed even tightly-matched queries capped around 0.80 cosine, so the old "‚â•0.80 = high" was unreachable. New thresholds: high ‚â•3 ‚â•0.75, medium ‚â•2 ‚â•0.70, low ‚â•1 ‚â•0.60, refuse 0 ‚â•0.60. **Refusal message** picks at random from **4 variants** so the demo doesn't feel scripted. Updated [[rag-query-pipeline]] (classifier gate, new thresholds, refusal variants, new SSE protocol), [[trellis-retrieval-implementation]] (full chat surface revision, new files table, conversation-wide citation resolution, stream-error compensation per commit `a8b4eb3`).

**Frontend protocol changes**: new `ChatKind = 'knowledge' | 'conversational'` type; `ChatMessage.kind` field; new `ChatTurn` type for history payloads; `chatStore.startStreaming(citedNodeIds, confidence, kind)` signature now takes kind, gates `overlayActive` to `knowledge` + non-refuse + non-empty citations; `chatStore.isPending` flag separates "submitted, awaiting first event" from "streaming." SSE consumer (`apps/web/src/api/chat.ts`) handles five event types: `kind`, `cited-nodes`, `token`, `done`, `error`.

**Shared prompt loader**: new `services/promptLoader.ts` with dev (tsx, `__dirname`) and prod (cwd `src/prompts/`) resolution; replaces the inline `readFileSync` previously in `rag.ts`. Used by both knowledge (`chat.md`) and conversational (`conversational.md`) paths.
